1 Introduction:

Wrangle, analyze and visualize the tweet archive of Twitter user @dog_rates , also known as WeRateDogs to create interesting and trustworthy analyses and visualizations using Jupyter Notebook.

2 Overview:

Real-world data is often dirty and messy. It rarely comes clean. The whole process of making this dirty and messy data usable for making quality analysis, producing descriptive visualizations and building predictive models is called Data Wrangling. It is done in three different steps gathering the required data, assessing it to identify the quality and structural issues in it and cleaning data.

 

In this project, I am using Python and its libraries to gather data from a variety of sources and in a variety of formats, assess it for its quality and tidiness, then finally clean it. I will also be documenting the wrangling efforts in a Jupyter Notebook and showcasing them through analysis and visualizations.

 

3 Data:

 

The dataset that is used in this project for wrangling, analyzing and visualizing is the tweet archive of Twitter user @dog_rates , also known as WeRateDogs. WeRateDogs is a Twitter account that rates people's dogs with a humorous comment about the dog. These ratings almost always have a denominator of 10 and the numerators almost always greater than 10 (Ex: 11/10, 12/10, 13/10, etc.) because they are good dogs. WeRateDogs has over 8 million followers and has received international media coverage.

The required data for this project is stored in different places:

 

3.1 Enhanced Twitter Archive:

 

The WeRateDogs Twitter archive contains basic tweet data for 5000+ of their tweets, but not everything. Though the column text for each tweets text in this archive contains enough information which was used to extract rating, dog name, and dog "stage" (i.e. doggo, floofer, pupper, and puppo) to make this Twitter archive "enhanced." There are only 2356 tweets with the dog ratings. This data is provided in twitter-archive-enhanced.csv file.

 

3.2 Image Predictions File:

 

This dataset is created by running every image in the WeRateDogs Twitter archive through a neural network that can classify breeds of dogs. This dataset contains top three predictions for each image alongside each tweet ID, image URL, and the image number that corresponded to the most confident prediction (numbered 1 to 4 since tweets can have up to four images).

This data is stored in file image_predictions.tsv which is hosted on Udacitys server. Tweet image prediction data is shown in the figure below:

The columns for showing results of top three image predictions are as follows:

 

         p1 is the algorithm's #1 prediction for the image in the tweet

         p1_conf is how confident the algorithm is in its #1 prediction

         p1_dog is whether or not the #1 prediction is a breed of dog

         p2 is the algorithm's second most likely prediction

         p2_conf is how confident the algorithm is in its #2 prediction

         p2_dog is whether or not the #2 prediction is a breed of dog

         p3 is the algorithm's second most likely prediction

         p3_conf is how confident the algorithm is in its #3 prediction

         p3_dog is whether or not the #3 prediction is a breed of dog

 

3.3 Additional Data via the Twitter API:

 

The initially provided Twitter archives just have the basic data regarding the tweets, so additional data like retweet count and favorite count for each tweet are needed for better analysis. Fortunately, this additional data can be gathered querying Twitter's API using the tweet IDs from original Twitter archives.

 

4 Key Points:

 

Key points to keep in mind when data wrangling for this project:

 

         Use only original ratings that have images not the retweets.

         The requirements of this project are only to assess and clean at least 8 quality issues and at least 2 tidiness issues in this dataset.

         Cleaning includes merging individual pieces of data according to the rules of tidy data.

         The fact that the rating numerators are greater than the denominators does not need to be cleaned. This unique rating system is a big part of the popularity of WeRateDogs.

         No need to gather the tweets beyond August 1st, 2017.

6 Gathering Data:

 

1.     Enhanced Twitter Archive:

 

  This dataset is provided in a file twitter-archive-enhanced.csv. I have manually downloaded and saved it in the working directory of this project.

  Then I used pandas read_csv() to import it in the project environment (Jupyter notebook wrangle_act.ipynb) in dataframe df_tae.

 

2.     Image Predictions File:

 

  Programmatically downloaded "image_predictions.tsv" which is hosted on Udacitys server using the Requests library and the following URL:

  " https://d17h27t6h515a5.cloudfront.net/topher/2017/August/599fd2ad_image-predictions/image-predictions.tsv"

  Created "image_predictions.tsv" file, requested the response from given url and wrote the response contents in "image_predictions.tsv".

  Imported the tsv file contents to pandas dataframe df_ip using read_csv() with parameter sep = \t.

 

3.     Additional Data via the Twitter API:

 

  Created an API object that can be used to gather Twitter data using tweepy library.

  Tweet data is stored in JSON format by Twitter.

  Queried Twitter API for each tweet in the Twitter archive and saved JSON in a text file.

  The tweets corresponding to a few tweet IDs in the archive, which was downloaded before, may have been deleted and twitters API may not have data related to them now. Used try-except block to handle this.

  To query all tweet IDs in the WeRateDogs Twitter archive takes around 20-30 minutes. Hence, printed tweet IDs after each query and put timer to check how much time needed. Also, set the wait_on_rate_limit and wait_on_rate_limit_notify parameters to True in get_status().

  Read tweet_json.txt file line by line to create a pandas dataframe df_tj with columns tweet_id, retweet_count and favorite_count.

 

7 Assessing Data:

 

  Assessed data in dataframes (df_tae, df_ip, df_tj) visually and programmatically for quality and tidiness issues.

  For visual assessment of all dataframes, I used Microsoft Excel.

  For programmatic assessment on df_tae, I used info(), describe(), groupby() with count(), value_counts(), duplicated() and various string manipulation functions.

  Saved the text and rating_numerator columns for rows where rating_denominator is 10, sorted by rating_numerator value in a csv file for more specific visual assessment.

  For programmatic assessment on df_ip, I used info(), describe(), value_counts() and duplicated().

  For programmatic assessment on df_tj, I used info() and duplicated().

8 Cleaning Data:

 

  Fixed the quality and tidiness issues identified in the assessment step.

  Made copy of all three dataframes before cleaning it.

 

8.1 Missing Data issues:

 

  There are lot of missing values in in_reply_to_status_id and in_reply_to_user_id columns as each tweet not necessarily have these two parameters. These parameters are not required for analyzing WeRateDogs data. Hence, dropped these columns.

  df_ip table has 281 missing records compare to df_tae table. This is because some tweets have posted videos not images. Hence, it is not possible to gather the related missing data.

  df_tj table has 25 missing records compare to df_tae table. Twitter archive data in df_tae is 3 years old. Some of the tweets from this dataset might have been deleted from twitter. Hence, there records are missing in recently queried data in df_tj. It is not possible to gather these records.

 

8.2 Tidiness issues:

 

  Created a single column "dog_stage" in df_tae to represent one of the four stages of dogs doggo, floofer, pupper and puppo, using their respective columns (doggo, floofer, pupper and puppo) and pandas apply(). Then dropped columns (doggo, floofer, pupper and puppo).

  Created a column "dog_breed" in df_ip for predicted dog breed, based on the confidence coefficient and prediction status(True or False) for all 3 given image predictions using apply(). Then dropped columns (p1, p1_conf, p1_dog, p2, p2_conf, p2_dog, p3, p3_conf, p3_dog). As p1 is the topmost prediction, used it first then p2 and finally p3.

  From summary statistics of df_ip table, the max values of p1_Conf, p2_Conf, p3_Conf are 1.0, 0.488, 0.2732 respectively. And for all rows in df_ip,

p1_Conf > p2_Conf > p3_conf

Hence, choose overall confidence level 0.2 for all three cases.

  df_ip should be part of df_tae. Hence, joined df_tae_clean and df_ip_clean on tweet_id column using pandas merge().

  df_tj also contains data related to tweet_ids in df_tae. Hence, joined them on tweet_id using pandas merge().

  Before merging, changed datatype of tweet_id column in df_tae table to string/object.

 

8.3 Quality issues:

 

df_tae:

  Retweet data in some rows indicates that these tweets are retweets. For this project, retweets are not required. Hence, deleted all the retweet records and columns with retweet related data.

  Changed datatype of column timestamp to datetime using to_datetime().

  Source urls are surronded in html tags. Extracted these urls using string split() function.

  Row with tweet_id (835246439529840000) was retweet. Hence, it was removed.

  For rating a dog's images, Denominator should be 10 as per WeRateDogs rating system. But, in some rare cases when there are multiple dogs in one image, they have given ratings like 80/80(for 8 dogs) or 60/50(for 5 dogs). For this project, I have ignored these few ratings and consider them invalid.

  Deleted all the ratings with denominator other than 10 (like 24/7, 1/2, 9/11, 4/20, 7/11, 11/15, 20/16 etc.)

  Some ratings have decimal values in text which was misinterpreted in ratings_numerator column. This error might have happen because of rating extraction from text field using regular expression "\d+/\d+".

  To solve this issue, extracted all the instances of decimal point ratings from the text column using regex search() and then replaced the erroneous ratings with these new ratings.

  Replaced string "None" with null values using numpy.nan.

  Removed all inaccurate lower case names in name column.

  Changed name "O" with "O'Malley" for tweet_id (776201521193218049).

  Finally, Stored cleaned and combined data in master dataset 'twitter_archive_master.csv'.
